# Typst 博客计划评审报告

**与当前计划对齐**：细粒度阶段见 [plan/schedule/00-index.md](../schedule/00-index.md)；后端与前端边界见 `01-backend-goals-and-scope.md` 与 `05-frontend-shell-and-routing.md`；**构建不依赖 npm、Hexo 式 CLI** 见 `07-hexo-like-cli-and-config.md`。

## 评审结论

项目可在 2~3 周上线，但若忽略 Typst HTML 能力验证、路径规范与 CI 一致性，容易出现返工并拉长到 4 周以上。

## 风险分级

### 高风险

- Typst HTML 导出能力与预期不一致（复杂排版、公式、脚注等）
- 资源与链接路径在部署后失效
- 本地与 CI Typst 版本不一致导致构建差异
- 部署非原子，失败时可能发布半成品

### 中风险

- 缺少统一元数据规范（title/date/summary）
- 构建脚本缺少失败处理与清理策略
- 未定义增量构建与草稿发布策略

### 低风险

- 首版样式一致性不足
- 高级 SEO 和搜索能力可后置

## 缺失任务补全

- 定义文章元数据字段与目录名（文章 id）规则
- 构建前清理策略与失败即终止策略
- 首页/列表页自动生成
- 链接完整性检查与最小回归测试
- README 与故障排查文档

## 工期建议

- 建议技术不确定性缓冲：20%~30%
- 每阶段设置硬验收点，不过线不进入下阶段
- 上线前预留 0.5~1 周回归窗口
