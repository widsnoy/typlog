# 04 后端：校验与 CI

## 校验（构建后）

在 **不启动浏览器自动化** 的前提下，至少做：

- **计数**：`post/` 中参与构建的文章数与 `public/posts/*.html` 数量一致（草稿规则见 02）。
- **重复文章 id**：若目录名冲突，构建失败。
- **可选**：简单 HTML 存在性检查（文件非空、包含 `<html` 或 `DOCTYPE`）。

## CI 工作流（后端阶段）

- 触发：`push` / `pull_request` 至主分支。
- 步骤：检出 → 安装 **固定版本** Typst → 运行构建（**非** `npm install` / `npm run`）→ 运行校验 → 失败则红；与 [07-hexo-like-cli-and-config.md](07-hexo-like-cli-and-config.md) 主入口一致。
- **本阶段可不部署**；若已有托管，可先 **artifact 上传** 或 **仅主分支部署**，按团队习惯。

## 版本锁定

- 在仓库记录：所需 Typst 精确版本（如 `0.14.x`）。
- CI 与 README 的安装方式一致（官方安装脚本、包管理器或容器镜像）。

## 验收清单

- [ ] 故意破坏一篇文章：CI 失败。
- [ ] 修复后：CI 通过。
- [ ] 无“仅本地通过”的隐藏步骤（所有步骤在 README 可复制）。

## 时间估算（参考）

- CI 初版：**0.5～1 天**
- 校验脚本：**0.5 天**
