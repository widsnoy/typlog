# 06 前端：质量、发布与扩展

**实现状态**：**未开始**（无全局 CSS、无 RSS/sitemap、README 无端到端部署路径）。

## 样式与无障碍（基础）

- 标题层级合理（页面级 `h1` 唯一或语义等价）。
- 对比度与字号可读；代码块可横向滚动。
- 不追求首版完美设计，但需 **无致命布局错误**（移动端可读）。

## SEO 与订阅（按需）

- `<title>`、`<meta name="description">` 最小集。
- RSS：`/rss.xml`（可在构建时由元数据生成）。
- `sitemap.xml`：文章 URL 列表。

## 部署与回滚

- 静态托管（GitHub Pages、Cloudflare Pages、对象存储等）任选；文档写明 **构建产物目录**。
- 失败策略：构建失败不覆盖线上；保留上一版本 artifact 或 tag 以便回滚。

## 扩展（排期）

- 标签页、全文搜索索引（JSON）、评论（第三方）——不在 MVP 强制范围。

## 整体验收（MVP 结束）

- [ ] 后端管线稳定（CI 绿）——待 04。
- [ ] 前端可浏览、无死链（抽样 + 列表全覆盖）。
- [ ] README：从克隆到本地构建、到部署的完整路径。

## 时间估算（参考）

- 打磨 + SEO/RSS 最小集：**1～3 天**
- 端到端文档与演练：**0.5～1 天**
